# Інструкція для контент-менеджерів ХНУРЕ

## Як працює сайт

Сайт побудований на технології **Astro** (Static Site Generation). Це означає:
- Сайт генерується як набір статичних HTML-файлів
- Контент підтягується з WordPress через API
- При зміні контенту в WordPress сайт автоматично перебудовується

### Переваги цього підходу
- **Швидкість**: сторінки завантажуються за <1 секунду
- **Безпека**: немає бази даних на фронтенді, неможливий SQL injection
- **Надійність**: статичні файли працюють навіть під навантаженням
- **SEO**: ідеальні Lighthouse-оцінки

## Публікація новин

### Через WordPress

1. Увійдіть в адмін-панель WordPress: `nure.ua/wp-admin`
2. Перейдіть: **Записи → Додати новий**
3. Заповніть:
   - **Заголовок** — короткий, інформативний
   - **Текст** — основний контент статті
   - **Уривок** — 1-2 речення для карточки на головній (поле "Excerpt")
   - **Зображення** — встановіть "Зображення запису" (рекомендації нижче)
   - **Категорія** — оберіть одну: Рейтинги / Наука / Події / Університет
4. Натисніть **Опублікувати**
5. Сайт автоматично перебудується через 2-5 хвилин

### Вимоги до зображень

| Тип | Розмір | Формат | Якість |
|-----|--------|--------|--------|
| Зображення новини | 800×400 px | WebP або JPG | 75-80% |
| Hero (головна) | 1920×1080 px | WebP | 80% |
| Логотипи партнерів | Будь-який | SVG | — |

**Конвертація в WebP:**
```bash
# macOS/Linux
cwebp -q 80 photo.jpg -o photo.webp

# Або використовуйте squoosh.app (безкоштовний онлайн-інструмент)
```

### Правила написання

- Заголовки: до 80 символів
- Уривки: 1-2 речення, до 160 символів
- Мова: офіційна, без сленгу
- Дати: формат «28 січня 2026»
- Посилання: повні URL (`https://...`)

## Управління спеціальностями

Дані спеціальностей зберігаються у файлі `src/lib/specialties.ts`. Для оновлення:

1. Відкрийте файл в будь-якому текстовому редакторі
2. Знайдіть потрібну спеціальність за кодом (121, 122, тощо)
3. Оновіть дані (опис, кар'єрні шляхи, предмети)
4. Збережіть файл
5. Зміни будуть видні після перебудови сайту

## Управління подіями

Файл: `src/lib/events.ts`

Формат події:
```typescript
{
  id: 'unique-id',
  title: { uk: 'Назва українською', en: 'English title' },
  date: '2026-04-19',        // формат YYYY-MM-DD
  endDate: '2026-04-20',     // якщо подія на кілька днів
  time: '10:00–14:00',       // необов'язково
  location: 'Корпус А',      // необов'язково
  type: 'science',           // academic | admission | science | culture | sport
  description: { uk: '...', en: '...' },
}
```

## Розгортання (deploy)

### Автоматичне (рекомендовано)
- Кожна зміна в WordPress → webhook → автоматична перебудова
- Кожен push в GitHub → CI/CD → автоматичний deploy

### Ручне
```bash
cd nure-site
npm run build        # генерація статичних файлів
npm run preview      # перевірка перед публікацією
```

## FAQ

**Q: Як швидко зміни з'являться на сайті?**
A: 2-5 хвилин після публікації в WordPress.

**Q: Чи можна редагувати сайт напряму?**
A: Ні. Контент редагується в WordPress, структура — через GitHub.

**Q: Що робити якщо сайт не оновлюється?**
A: Перевірте webhook у WordPress Settings → NURE Webhook. Або зв'яжіться з розробником.

**Q: Як додати нову сторінку?**
A: Зв'яжіться з розробником. Нові сторінки потребують створення шаблону.

**Q: Підтримка мобільних пристроїв?**
A: Так, сайт повністю адаптивний (responsive). Тестується на iOS та Android.
